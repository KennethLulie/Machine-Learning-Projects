Analysis of Low Birthweight Children TODO - LINK TABLE OF CONENTS AS IN EXAMPLE¶

Using Logicistic Regression to Review the factors of and predict birthweight of newly born children. The predictors included in the data are the mothers age, the Race, if the mother Smokes, if the mother has a history of premature labor, if the mother has a history of hypertension, if the mother has Uterine irritability, and the number of physician visits during the first semester.

The target variable is the birth weight of the baby with low being considered under 2500g (1), and high being over 2500g (0). For reference, 2,500 is 2.5kg or around 5 and 1/2 pounds.

Dataset source¶

Hosmer, D.W., Lemeshow, S. and Sturdivant, R.X. (2013) Applied Logistic Regression, 3rd ed., New York: Wiley This dataset is also part of the aplore3 R package.

Dataset fields¶

ID Identification Code

LOW Low birth weight (0: >= 2500, 1: < 2500 g) - target variable

AGE Mother’s age in years

RACE Race (1: White, 2: Black, 3: Other)

SMOKE Smoking status during pregnancy (1: No, 2: Yes)

PTL History of premature labor (1: None, 2: One, 3: Two, etc)

HT History of hypertension (1: No, 2: Yes)

UI Presence of Uterine irritability (1: No, 2: Yes)

FTV Number of physician visits during the first trimester (1: None, 2: One, 3: Two, etc)

# Label is a target variable. PersonInfo is a list of independent variables besides unique identifier

LabeledDocument = Row("ID", "MotherInfo", "label")

# Define a function that parses each raw in CSV file and returns an object of type LabeledDocument

def parseDocument(line):
    values = [str(x) for x in line.split(',')] 
    if (values[1]>'0'):
      LOW = 1.0
    else:
      LOW = 0.0
#Index 2 is Age, Index 3 is Race, Index 4 is Smoking Status, and Index 8 is physician visits in first trimester
    textValue = str(values[2]) + " " + str(values[3])+" " + str(values[4])+" "+ str(values[8])
    return LabeledDocument(values[5], textValue, LOW)

Construction of Logistic Regression Model¶

Using SPARKMl pipeline to build a standard logistic regression model

Tokenizer is a unary transformer (applies a transformation to one input column and writes result to another by creating a new column Specifically it converts the column of string values to lowercase and then splits it by white spaces. For example

id |sentence |tokens | +---+-----------------------------+-----------------------------------+ |1 |Hello world! |[hello, world!] | |2 |Here is yet another sentence.|[here, is, yet, another, sentence.]|

More details can be found here --> https://jaceklaskowski.gitbooks.io/mastering-apache-spark/spark-mllib/spark-mllib-transformers-Tokenizer.html

hasingTF - Is an efficient way to hash the data to reduce memory and computational overhead.

LogisticRegression - calling the sparkML library for running logistic regression. We have it set to a maximum number of iterations at 10,and the reg param set to 0.01.

For reference, 0 regparam reduces the model to a ridge regression model, and 1 is a lasso model. For more information go to this link https://spark.apache.org/docs/1.5.2/ml-linear-methods.html

In [34]:
tokenizer = Tokenizer(inputCol="MotherInfo", outputCol="words")
hashingTF = HashingTF(inputCol=tokenizer.getOutputCol(), outputCol="features")
lr = LogisticRegression(maxIter=100, regParam=0.01)
pipeline = Pipeline(stages=[tokenizer, hashingTF, lr])
